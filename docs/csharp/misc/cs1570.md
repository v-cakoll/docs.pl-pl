# <a name="compiler-warning-level-1-cs1570"></a>Kompilatora (poziom 1) ostrzeżenie CS1570
Komentarz XML w elemencie "konstrukcji" został nieprawidłowo sformułowany kod XML — "reason"  
  
 Korzystając z [/doc](../../csharp/language-reference/compiler-options/doc-compiler-option.md), wszelkie komentarze w kodzie źródłowym musi być w formacie XML. Wszelkie błędy przy użyciu znaczników XML wygeneruje CS1570. Na przykład:  
  
- Jeśli przekazujesz ciąg **cref**, na przykład [ \<wyjątek >](../../csharp/programming-guide/xmldoc/exception.md) tagu ciągu muszą być ujęte w podwójny cudzysłów.  
  
- Jeśli używasz znacznik, taki jak [ \<SeeAlso — >](../../csharp/programming-guide/xmldoc/seealso.md), nie ma tagu zamykającego, należy określić ukośnik przed nawias zamykający.  
  
- Jeśli musisz użyć — lub mniejsze-niż symbol tekst opisu, należy do reprezentowania ich za pomocą **&gt;** lub **&lt;**.  
  
- Atrybut pliku lub ścieżkę na [ \<obejmują >](../../csharp/programming-guide/xmldoc/include.md) tag brakowało lub nieprawidłowo sformułowany.  
  
 Poniższy przykład spowoduje wygenerowanie CS1570:  
  
```csharp  
// CS1570.cs  
// compile with: /W:1  
namespace ns  
{  
   // the following line generates CS1570  
   /// <summary> returns true if < 5 </summary>  
   // try this instead  
   // /// <summary> returns true if <5 </summary>  
  
   public class MyClass  
   {  
      public static void Main ()  
      {  
      }  
   }  
}  
```
