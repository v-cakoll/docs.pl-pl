# <a name="compiler-error-cs0012"></a>CS0012 błąd kompilatora
Typ "type" jest zdefiniowany w zestawie, do którego nie odwołuje się. Należy dodać odwołanie do zestawu "assembly".  
  
 Nie znaleziono definicji typu występującego w odwołaniu. Może to wystąpić, jeśli jest to wymagane. Plik DLL nie jest uwzględniony w kompilacji. Aby uzyskać więcej informacji, zobacz [odwołania okno dialogowe Dodawanie](http://msdn.microsoft.com/en-us/2feb0fe2-0805-4cc9-8cba-b0315849dfb7) i [/Reference (opcje kompilatora C#)](../../csharp/language-reference/compiler-options/reference-compiler-option.md).  
  
 Następująca sekwencja kompilacje spowoduje CS0012:  
  
```  
// cs0012a.cs  
// compile with: /target:library  
public class A {}  
```  
  
 Następnie:  
  
```  
// cs0012b.cs  
// compile with: /target:library /reference:cs0012a.dll  
public class B  
{  
   public static A f()  
   {  
      return new A();  
   }  
}  
```  
  
 Następnie:  
  
```  
// cs0012c.cs  
// compile with: /reference:cs0012b.dll  
class C  
{  
   public static void Main()  
   {  
      object o = B.f();   // CS0012  
   }  
}  
```  
  
 CS0012 ten można rozwiązać przez kompilowania przy użyciu `/reference:cs0012b.dll;cs0012a.dll`, lub w programie Visual Studio za pomocą [odwołania okno dialogowe Dodawanie](http://msdn.microsoft.com/en-us/2feb0fe2-0805-4cc9-8cba-b0315849dfb7) można dodać odwołania do cs0012a.dll oprócz cs0012b.dll.
