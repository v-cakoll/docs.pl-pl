 <span data-ttu-id="35345-108">W usłudze operacja pobiera tablicę liczb całkowitych przekazaną w komunikacie wejściowym, a następnie oblicza sumę.</span><span class="sxs-lookup"><span data-stu-id="35345-108">On the service, the operation retrieves the array of integers passed in the input message and then computes the sum.</span></span> <span data-ttu-id="35345-109">Aby wysłać komunikat odpowiedzi, przykład tworzy nową wiadomość z odpowiednią wersją i akcją wiadomości i dodaje obliczoną sumę jako treść.</span><span class="sxs-lookup"><span data-stu-id="35345-109">To send a response message, the sample creates a new message with the appropriate message version and Action and adds the computed sum as its body.</span></span> <span data-ttu-id="35345-110">Poniższy przykładowy kod demonstruje ten sposób.</span><span class="sxs-lookup"><span data-stu-id="35345-110">The following sample code demonstrates this.</span></span>  
  
```csharp
public Message ComputeSum(Message request)  
{  
    //The body of the message contains a list of numbers which will be
    //read as a int[] using GetBody<T>  
    int result = 0;  
  
    int[] inputs = request.GetBody<int[]>();  
    foreach (int i in inputs)  
    {  
        result += i;  
    }  
  
    Message response = Message.CreateMessage(request.Version,
                                      ReplyAction, result);  
    return response;  
}  
```  
  
 <span data-ttu-id="35345-111">Klient korzysta z kodu wygenerowanego przez [Narzędzie do obsługi metadanych ServiceModel (Svcutil. exe)](../servicemodel-metadata-utility-tool-svcutil-exe.md) , aby wersję wiadomości, która zależy od kanału bazowego.</span><span class="sxs-lookup"><span data-stu-id="35345-112">To send a request message, the client must have the message version, which depends on data-stu-id="35345-113">Thus, it creates a new <xref:System.ServiceModel.OperationContextScope> scoped to the proxy channel it created, which creates an <xref:System.ServiceModel.OperationContext> żądania, a następnie wywoła `ComputeSum` na serwerze proxy.</span><span class="sxs-lookup"><span data-stu-id="35345-114">The client passes an input array as the body to the request message and then invokes the `ComputeSum` on the proxy.</span></span> <span data-ttu-id="35345-115">Następnie klient pobiera sumę danych wejściowych, które przekazuje, uzyskując dostęp do `GetBody<T>` metody w komunikacie odpowiedzi.</span><span class="sxs-lookup"><span data-stu-id="35345-115">The client then retrieves the sum of the inputs it passed by accessing the `GetBody<T>` method on the reply message.</span></span> <span data-ttu-id="35345-116">Poniższy przykładowy kod demonstruje ten sposób.</span><span class="sxs-lookup"><span data-stu-id="35345-116">The following sample code demonstrates this.</span></span>  
  
```csharp
using (new OperationContextScope(client.InnerChannel))  
{  
    // Call the Sum service operation.  
    int[] values = { 1, 2, 3, 4, 5 };  
    Message request = Message.CreateMessage(  
        OperationContext.Current.OutgoingMessageHeaders.MessageVersion,
        RequestAction, values);  
    Message reply = client.ComputeSum(request);  
    int response = reply.GetBody<int>();  
  
    Console.WriteLine("Sum of numbers passed (1,2,3,4,5) = {0}",
                                                       response);  
}  
```
