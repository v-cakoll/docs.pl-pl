## <a name="using-managed-wrapper-classes"></a><span data-ttu-id="53425-179">Używanie zarządzanych klas otoki</span><span class="sxs-lookup"><span data-stu-id="53425-179">Using Managed Wrapper Classes</span></span>

<span data-ttu-id="53425-180">Większość aplikacji i składników (z wyjątkiem bezpiecznych bibliotek) nie powinna bezpośrednio wywołać kodu niezarządzanego.</span><span class="sxs-lookup"><span data-stu-id="53425-180">Most applications and components (except secure libraries) should not directly call unmanaged code.</span></span> <span data-ttu-id="53425-181">Istnieje kilka przyczyn tego.</span><span class="sxs-lookup"><span data-stu-id="53425-181">There are several reasons for this.</span></span> <span data-ttu-id="53425-182">Jeśli kod niezarządzany jest bezpośrednio wywoływany, nie będzie można go uruchamiać w wielu przypadkach, ponieważ kod musi mieć przydzielony wysoki poziom zaufania, aby wywołać kod natywny.</span><span class="sxs-lookup"><span data-stu-id="53425-182">If code calls unmanaged code directly, it will not be allowed to run in many circumstances because code must be granted a high level of trust to call native code.</span></span> <span data-ttu-id="53425-183">Jeśli zasady są modyfikowane w celu zezwolenia na uruchomienie takiej aplikacji, może znacząco obniżyć poziom bezpieczeństwa systemu, pozostawiając aplikację bezpłatnie do wykonania niemal każdej operacji.</span><span class="sxs-lookup"><span data-stu-id="53425-183">If policy is modified to allow such an application to run, it can significantly weaken the security of the system, leaving the application free to perform almost any operation.</span></span>
