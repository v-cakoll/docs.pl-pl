<span data-ttu-id="e5304-104">Zgodność z poprzednimi wersjami oznacza, że aplikacja opracowana dla konkretnej wersji platformy będzie działać w nowszych wersjach tej platformy.</span><span class="sxs-lookup"><span data-stu-id="e5304-104">Backward compatibility means that an app that was developed for a particular version of a platform will run on later versions of that platform.</span></span> <span data-ttu-id="e5304-105">.NET Framework próbuje zmaksymalizować zgodność z poprzednimi wersjami: kod źródłowy zapisany dla jednej wersji .NET Framework powinien zostać skompilowany w nowszych wersjach .NET Framework, a pliki binarne, które działają w jednej wersji .NET Framework, powinny zachowywać się identycznie w nowszych wersjach .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="e5304-105">.NET Framework tries to maximize backward compatibility: Source code written for one version of .NET Framework should compile on later versions of .NET Framework, and binaries that run on one version of .NET Framework should behave identically on later versions of .NET Framework.</span></span>

## <a name="version-compatibility-for-apps"></a><a name="Apps"></a><span data-ttu-id="e5304-106">Zgodność wersji aplikacji</span><span class="sxs-lookup"><span data-stu-id="e5304-106">Version compatibility for apps</span></span>

<span data-ttu-id="e5304-110">Aby zdefiniować konkretne wersje, na których działa aplikacja, Dodaj co najmniej jeden [\<supportedRuntime>](../configure-apps/file-schema/startup/supportedruntime-element.md) element do pliku konfiguracji aplikacji.</span><span class="sxs-lookup"><span data-stu-id="e5304-110">To define the specific versions on which your app runs, add one or more [\<supportedRuntime>](../configure-apps/file-schema/startup/supportedruntime-element.md) elements to your app's configuration file.</span></span> <span data-ttu-id="e5304-111">Każdy `<supportedRuntime>` element zawiera obsługiwaną wersję środowiska uruchomieniowego, z pierwszym określeniem najbardziej preferowanej wersji i ostatnim określeniem najmniejszej wersji.</span><span class="sxs-lookup"><span data-stu-id="e5304-111">Each `<supportedRuntime>` element lists a supported version of the runtime, with the first specifying the most preferred version and the last specifying the least preferred version.</span></span>

```xml
   <startup>
      <supportedRuntime version="v2.0.50727" />
      <supportedRuntime version="v4.0" />
   </startup>
</configuration>
```

<span data-ttu-id="e5304-120">.NET Framework 4,5 i nowsze wersje są zgodne wstecz z aplikacjami, które zostały skompilowane ze starszymi wersjami .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="e5304-120">The .NET Framework 4.5 and later versions are backward-compatible with apps that were built with earlier versions of the .NET Framework.</span></span> <span data-ttu-id="e5304-121">Innymi słowy aplikacje i składniki utworzone przy użyciu poprzednich wersji będą działały bez modyfikacji w .NET Framework 4,5 i nowszych wersjach.</span><span class="sxs-lookup"><span data-stu-id="e5304-121">In other words, apps and components built with previous versions will work without modification on the .NET Framework 4.5 and later versions.</span></span> <span data-ttu-id="e5304-122">Jednak domyślnie aplikacje są uruchamiane w wersji środowiska uruchomieniowego języka wspólnego, dla którego zostały opracowane, więc może być konieczne dostarczenie pliku konfiguracji w celu umożliwienia uruchomienia aplikacji w .NET Framework 4,5 lub nowszych.</span><span class="sxs-lookup"><span data-stu-id="e5304-122">However, by default, apps run on the version of the common language runtime for which they were developed, so you may have to provide a configuration file to enable your app to run on the .NET Framework 4.5 or later versions.</span></span> <span data-ttu-id="e5304-123">Aby uzyskać więcej informacji, zobacz sekcję [zgodność wersji dla aplikacji](#Apps) wcześniej w tym artykule.</span><span class="sxs-lookup"><span data-stu-id="e5304-123">For more information, see the [Version compatibility for apps](#Apps) section earlier in this article.</span></span>
