# <a name="compiler-error-cs0011"></a>Błąd kompilatora CS0011
Nie można rozpoznać klasy bazowej lub interfejsu "Class" w zestawie "Assembly", do którego odwołuje się typ "Type"  
  
 Klasa, która została zaimportowana z pliku z **/Reference**, pochodzi od klasy lub implementuje interfejs, który nie został odnaleziony. Taka sytuacja może wystąpić, jeśli wymagana biblioteka DLL nie jest również dołączana do kompilacji z **/Reference**.  
  
 Aby uzyskać więcej informacji, zobacz [okno dialogowe Dodawanie odwołania](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) i [odwołanie (C# opcje kompilatora)](../language-reference/compiler-options/reference-compiler-option.md).  
  
## <a name="example"></a>Przykład  
  
```csharp  
// CS0011_1.cs  
// compile with: /target:library  
  
public class Outer   
{  
   public class B { }  
}  
```  
  
## <a name="example"></a>Przykład  
 Drugi plik tworzy bibliotekę DLL, która definiuje klasę `C`, która pochodzi od klasy `B`, która została utworzona w poprzednim przykładzie.  
  
```csharp  
// CS0011_2.cs  
// compile with: /target:library /reference:CS0011_1.dll  
// post-build command: del /f CS0011_1.dll  
public class C : Outer.B {}  
```  
  
## <a name="example"></a>Przykład  
 Trzeci plik zastępuje bibliotekę DLL utworzoną przez pierwszy krok i pomija definicję wewnętrznej klasy `B`.  
  
```csharp  
// CS0011_3.cs  
// compile with: /target:library /out:cs0011_1.dll  
public class Outer {}  
```  
  
## <a name="example"></a>Przykład  
 Na koniec czwarty plik odwołuje się do klasy `C` zdefiniowanej w drugim przykładzie, który jest pochodną klasy `B`, która nie istnieje.  
  
 Poniższy przykład generuje CS0011.  
  
```csharp  
// CS0011_4.cs  
// compile with: /reference:CS0011_1.dll /reference:CS0011_2.dll  
// CS0011 expected  
  
class M  
{  
   public static void Main()  
   {  
      C c = new C();  
   }  
}  
```
