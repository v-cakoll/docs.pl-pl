### <a name="enable-security-with-peer-channel-applications"></a><span data-ttu-id="ded47-119">Włącz zabezpieczenia za pomocą aplikacji kanału równorzędnego</span><span class="sxs-lookup"><span data-stu-id="ded47-119">Enable Security with Peer Channel Applications</span></span>  
 <span data-ttu-id="ded47-120">Ze względu na rozproszony charakter protokołów kanałów równorzędnych trudno jest wymusić członkostwo w sieci, poufność i prywatność w niezabezpieczonej sieci.</span><span class="sxs-lookup"><span data-stu-id="ded47-120">Due to the distributed nature of the Peer Channel protocols, it is hard to enforce mesh membership, confidentiality, and privacy in an unsecured mesh.</span></span> <span data-ttu-id="ded47-121">Należy również pamiętać o zabezpieczeniu komunikacji między klientami a usługą rozpoznawania nazw.</span><span class="sxs-lookup"><span data-stu-id="ded47-121">It is also important to remember to secure communication between clients and the resolver service.</span></span> <span data-ttu-id="ded47-122">W obszarze Protokół rozpoznawania nazw równorzędnych (PNRP) używaj bezpiecznych nazw, aby uniknąć fałszowania i innych typowych ataków.</span><span class="sxs-lookup"><span data-stu-id="ded47-122">Under Peer Name Resolution Protocol (PNRP), use secure names to avoid spoofing and other common attacks.</span></span> <span data-ttu-id="ded47-123">Zabezpiecz niestandardową usługę programu rozpoznawania nazw, włączając zabezpieczenia na klientach połączenia używane do kontaktowania się z usługą rozpoznawania nazw, w tym zabezpieczeniami komunikatów i opartych na transportach.</span><span class="sxs-lookup"><span data-stu-id="ded47-123">Secure a custom resolver service by enabling security on the connection clients use to contact the resolver service, including both message- and transport-based security.</span></span>

 <span data-ttu-id="ded47-125">Na przykład, jeśli każdy element członkowski siatki musi być zidentyfikowany indywidualnie, użyj modelu uwierzytelniania opartego na certyfikatach.</span><span class="sxs-lookup"><span data-stu-id="ded47-125">For example, if each member of the mesh needs to be individually identified, use certificate-based authentication model.</span></span> <span data-ttu-id="ded47-126">Jeśli nie jest to możliwe, należy użyć uwierzytelniania opartego na hasłach zgodnie z bieżącymi zaleceniami, aby zapewnić ich bezpieczeństwo.</span><span class="sxs-lookup"><span data-stu-id="ded47-126">If that is not possible, use password-based authentication following current recommendations to keep them secure.</span></span> <span data-ttu-id="ded47-127">Dotyczy to również udostępniania haseł tylko zaufanym podmiotom, przesyłania haseł przy użyciu bezpiecznego nośnika, często zmieniających się haseł i zapewniania silnych haseł (co najmniej osiem znaków).</span><span class="sxs-lookup"><span data-stu-id="ded47-127">This includes sharing passwords only with trusted parties, transmitting passwords using a secure medium, changing passwords frequently, and ensuring that passwords are strong (at least eight characters long, include at least one letter from both cases, a digit, and a special character).</span></span>
