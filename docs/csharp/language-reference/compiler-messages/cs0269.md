# <a name="compiler-error-cs0269"></a>Błąd kompilatora CS0269
Użycie nieprzypisanego parametru "parametru" out  
  
 Kompilator nie może zweryfikować, czy parametr out została przypisana wartość, zanim został on użyty; wartość może być Niezdefiniowany, po przypisaniu. Pamiętaj przypisać wartość do `out` parametrów w metodzie wywoływanej przed uzyskaniem dostępu do wartości. Jeśli chcesz użyć wartości zmiennej przekazanej, użyj `ref` parametru zamiast tego. Aby uzyskać więcej informacji, zobacz [przekazanie parametrów](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).  
  
## <a name="example"></a>Przykład  
 Poniższy przykład spowoduje wygenerowanie CS0269:  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a>Przykład  
 To może również wystąpić, jeśli wystąpi inicjowania zmiennej w bloku try, czyli kompilator nie można zweryfikować wykona pomyślnie:  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
