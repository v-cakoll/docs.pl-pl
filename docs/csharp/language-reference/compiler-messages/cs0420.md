# <a name="compiler-warning-level-1-cs0420"></a>Ostrzeżenie kompilatora (poziom 1) CS0420
"Identyfikator": odwołanie do pola nietrwałego nie będzie traktowane jako nietrwałe  
  
 Pole nietrwałe nie powinno być zwykle przenoszone przy `ref` użyciu parametru lub **out** , ponieważ nie będzie traktowane jako nietrwały w zakresie funkcji. Istnieją między innymi wyjątki, takie jak podczas wywoływania interfejsu API z blokadą. Podobnie jak w przypadku dowolnego ostrzeżenia, można użyć [ostrzeżenia #pragma](../preprocessor-directives/preprocessor-pragma-warning.md) , aby wyłączyć to ostrzeżenie w rzadkich przypadkach, gdy celowo używasz pola nietrwałego jako parametru odwołania.  
  
 Poniższy przykład generuje CS0420:  
  
```csharp  
// CS0420.cs  
// compile with: /W:1  
using System;  
  
class TestClass  
{  
   private volatile int i;  
  
   public void TestVolatile(ref int ii)  
   {  
   }  
  
   public static void Main()  
   {  
      TestClass x = new TestClass();  
      x.TestVolatile(ref x.i);   // CS0420   
   }  
}  
```
