# <a name="compiler-error-cs0246"></a>Błąd kompilatora CS0246
Nie można znaleźć nazwy typu lub przestrzeni nazw "Type/namespace" (czy nie brakuje dyrektywy using lub odwołania do zestawu?)  
  
 Nie znaleziono typu lub przestrzeni nazw użytej w programie. Być może odwołano się do odwołania ([-Reference](../compiler-options/reference-compiler-option.md)) zestawu, który zawiera typ, lub nie dodano wymaganej [dyrektywy using](../keywords/using-directive.md).  Może wystąpić problem z zestawem, do którego próbujesz się odwołać.  
  
 Następujące sytuacje powodują wystąpienie błędu kompilatora CS0246.  
  
- Czy poznasz nazwę typu lub przestrzeni nazw? Bez poprawnej nazwy kompilator nie może odnaleźć definicji typu lub przestrzeni nazw. Często dzieje się tak, ponieważ wielkość liter użyta w nazwie typu jest niepoprawna. Na przykład `Dataset ds;` generuje CS0246, ponieważ muszą być pisane wielkimi literami "s in DataSet".  
  
- Jeśli błąd dotyczy nazwy przestrzeni nazw, dowiesz się, jak dodać odwołanie ([-odwołanie](../compiler-options/reference-compiler-option.md)) do zestawu, który zawiera przestrzeń nazw? Na przykład kod może zawierać dyrektywę `using Accessibility`. Jeśli jednak projekt nie odwołuje się do biblioteki Accessibility. dll, zgłaszany jest błąd CS0246. Aby uzyskać więcej informacji, zobacz [Zarządzanie odwołaniami w projekcie](/visualstudio/ide/managing-references-in-a-project)  
  
- Jeśli błąd dotyczy nazwy typu, podano odpowiednią [dyrektywę using](../keywords/using-directive.md)lub, Alternatywnie, w pełni zakwalifikowanie nazwy typu? Rozważmy następującą deklarację: `DataSet ds`. Aby użyć typu `DataSet`, potrzebne są dwie rzeczy. Najpierw należy odwołać się do zestawu, który zawiera definicję dla typu `DataSet`. Po drugie należy `using` dyrektywę dla przestrzeni nazw, w której znajduje się `DataSet`. Na przykład, ponieważ `DataSet` znajduje się w przestrzeni nazw **System. Data** , na początku kodu będzie potrzebna następująca dyrektywa: `using System.Data`.  
  
     Dyrektywa `using` nie jest wymagana. Jednakże w przypadku pominięcia dyrektywy należy w pełni zakwalifikować typ `DataSet` w przypadku odwoływania się do niego. Pełna kwalifikacja oznacza, że należy określić zarówno przestrzeń nazw, jak i typ za każdym razem, gdy użytkownik odwołuje się do typu w kodzie. W przypadku pominięcia dyrektywy `using` w poprzednim przykładzie należy napisać `System.Data.DataSet ds`, aby zadeklarować `ds` zamiast `DataSet ds`.  
  
- Czy użyto zmiennej lub innego elementu języka, w którym oczekiwano typu? Na przykład w instrukcji **is** , jeśli używasz obiektu `Type` zamiast rzeczywistego typu, zostanie wyświetlony komunikat o błędzie CS0246.  

- Czy odwołano się do zestawu, który został zbudowany względem nowszej wersji platformy niż platforma docelowa programu? Czy odwołano się do projektu, który jest przeznaczony dla wyższej wersji platformy niż docelowa platforma programu? Na przykład pracujesz nad projektem, który jest przeznaczony dla .NET Framework 4.6.1 i użyj typu z projektu, który jest przeznaczony dla .NET Framework 4.7.1. Następnie zostanie wyświetlony komunikat o błędzie CS0246.
  
- Czy używasz *dyrektywy aliasu using* bez w pełni kwalifikowanej nazwy typu? Dyrektywa aliasu `using` nie używa dyrektyw `using` w pliku kodu źródłowego do rozpoznawania typów. Poniższy przykład generuje CS0246, ponieważ typ `List<int>` nie jest w pełni kwalifikowany. Dyrektywa `using` dla `System.Collections.Generic` nie zapobiega wystąpieniu błędu.  
  
    ```csharp  
    using System.Collections.Generic;  
  
    // The following declaration generates CS0246.  
    using myAliasName = List<int>;   
  
    // To avoid the error, fully qualify List.  
    using myAliasName2 = System.Collections.Generic.List<int>;  
    ```  
  
     Jeśli ten błąd występuje w kodzie, który był wcześniej działał, należy najpierw wyszukać brakujące lub nierozwiązane odwołania w Eksplorator rozwiązań. Czy musisz ponownie zainstalować pakiet [NuGet](https://www.nuget.org/) ? Aby uzyskać informacje na temat sposobu wyszukiwania odwołań przez system kompilacji, zobacz [rozpoznawanie odwołań do plików w kompilacji zespołu](https://docs.microsoft.com/archive/blogs/manishagarwal/resolving-file-references-in-team-build-part-2). Jeśli wszystkie odwołania wydają się być poprawne, poszukaj w historii kontroli źródła, aby zobaczyć, co zmieniło się w pliku. csproj i/lub w lokalnym pliku źródłowym.  
  
     Jeśli nie masz jeszcze dostępu do odwołania, użyj Przeglądarka obiektów do sprawdzenia zestawu, który powinien zawierać tę przestrzeń nazw, i sprawdź, czy przestrzeń nazw jest obecna. Jeśli sprawdzasz, Przeglądarka obiektów, że zestaw zawiera przestrzeń nazw, spróbuj usunąć dyrektywę `using` dla przestrzeni nazw i zobaczyć, co inne przerwy. Problem z katalogiem głównym może mieć inny typ w innym zestawie.  
  
 Poniższy przykład generuje CS0246, ponieważ brakuje wymaganej dyrektywy `using`.  
  
```csharp  
// CS0246.cs  
//using System.Diagnostics;  
  
public class MyClass  
{  
    // The following line causes CS0246. To fix the error, uncomment  
    // the using directive for the namespace for this attribute,  
    // System.Diagnostics.  
    [Conditional("A")]  
    public void Test()  
    {  
    }  
  
    public static void Main()  
    {  
    }  
}  
```  
  
 Poniższy przykład powoduje wystąpienie CS0246, ponieważ został użyty obiekt typu `Type`, w którym oczekiwano rzeczywistego typu.  
  
```csharp  
// CS0246b.cs  
using System;  
  
class ExampleClass  
{  
    public bool supports(object o, Type t)  
    {  
        // The following line causes CS0246. You must use an  
        // actual type, such as ExampleClass, String, or Type.  
        if (o is t)  
        {  
            return true;  
        }  
        return false;  
    }  
}  
  
class Program  
{  
    public static void Main()  
    {  
        ExampleClass myC = new ExampleClass();  
        myC.supports(myC, myC.GetType());  
    }  
}  
```
